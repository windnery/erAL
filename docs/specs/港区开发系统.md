# 港区开发系统

## 概述

港区开发系统使用港区资金升级设施，目标是提升互动体验相关收益，而不是做重度经营模拟。

## 当前状态

- 当前未实现独立港区开发系统。
- 地图和地点标签已存在，可作为设施挂载位置。

## 数据结构

### FacilityDefinition

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 设施 key |
| `display_name` | str | 设施名称 |
| `max_level` | int | 最大等级 |
| `upgrade_costs` | [int] | 各级升级费用 |
| `effects` | {str: int} | 升级效果 |

### FacilityState

| 字段 | 类型 | 说明 |
|------|------|------|
| `facility_key` | str | 设施 key |
| `level` | int | 当前等级 |

## 建议首批设施

- `dorm`：提高休息恢复效率。
- `canteen`：提高工作与委托的基础收益。
- `date_spot`：提高约会类互动的关系收益。

## 流程

1. 玩家选择设施升级。
2. 检查港区资金是否足够。
3. 扣除港区资金并提升设施等级。
4. 将效果广播给体力恢复、收益或关系结算。

## 与现有系统关系

- 消费走 [资金与账本系统](资金与账本系统.md)。
- 恢复加成作用于 [体力与气力系统](体力与气力系统.md)。
- 关系加成作用于 [关系系统](关系系统.md) 或 [结算系统](结算系统.md)。

## 字段对齐说明

- 设施状态建议挂在世界层，命名使用 `facility_levels`（`dict[str, int]`）即可覆盖第一版需求。
- 不建议第一版引入复杂对象图，先用 key-level 映射保持与当前项目数据风格一致。
- 所有设施效果建议最终映射到现有结算点：恢复倍率、收益倍率、关系增益。
