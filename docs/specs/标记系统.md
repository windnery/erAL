# 标记系统

## 概述

标记（Mark）是附在角色身上的状态标签，用于条件分支。分为两类：持久标记（如 confessed、kissed）记录经历，状态标记（如 drunk、embarrassed、angry）反映当前状态。标记参与指令可用性、事件触发、口上选择三个条件匹配环节。

## 数据结构

### MarkDefinition（标记定义）

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 唯一标识 |
| `display_name` | str | 显示名称 |
| `group` | str | 分类：relationship/intimacy/condition |
| `max_level` | int | 上限等级 |

### CharacterState.marks

`dict[str, int]`——键为标记 key，值为当前等级。缺失的标记视为等级 0。

## 流程

### 施加标记

```
1. 指令执行后，CommandService._apply_marks() 遍历 command.apply_marks
2. 查找 MarkDefinition 获取 max_level（未定义则默认 1）
3. actor.add_mark(key, delta, max_level) — 增量施加，clamp 到 [0, max_level]
```

### 标记条件匹配

三个系统使用相同的匹配逻辑：

```python
for mark_key, min_level in required_marks.items():
    if scene.marks.get(mark_key, 0) < min_level:
        return False  # 不满足
```

- **指令可用性**：`command.required_marks` → 角色必须持有指定等级的标记
- **事件触发**：`event.required_marks` → 场景中角色标记满足条件
- **口上选择**：`entry.required_marks` → 同上

### 场景上下文传递

```
SceneService.build_for_actor() 从 actor.marks 复制到 SceneContext.marks
→ EventService / DialogueService 使用 scene.marks 做条件匹配
```

### 标记施加

在 `data/base/commands.toml` 的 `apply_marks` 字段中定义：

| 指令 | 施加标记 |
|------|---------|
| serve_tea | drunk +1 |
| scold | angry +1 |
| tease | teased +1, embarrassed +1 |
| date_tease | teased +1, embarrassed +1 |
| kiss | kissed +1 |
| confess | confessed +1 |

## 扩展方法

1. **加新标记**：在 `marks.toml` 中加 `[[marks]]`，定义 key/display_name/group/max_level
2. **指令施加标记**：在 `commands.toml` 的指令中加 `apply_marks = { 新标记 = 1 }`
3. **标记作为条件**：在指令（`required_marks`）、事件（`required_marks`）、口上（`required_marks`）中使用
4. **调整上限**：修改 `max_level`，如 teased=3 意味着调戏 3 次后不再叠加
