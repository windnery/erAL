# erAL 系统规格文档

每个文档覆盖一个子系统，结构统一：**数据结构 → 流程 → 配置 → 扩展方法**。

## 文档列表

| 文档 | 说明 |
|------|------|
| [结算系统](结算系统.md) | SOURCE → BASE/PALAM/CFLAG/TFLAG 的统一结算管道 |
| [指令系统](指令系统.md) | 玩家可用指令的定义、过滤和执行流程 |
| [标记系统](标记系统.md) | 角色状态标记的施加、条件匹配和上限控制 |
| [事件系统](事件系统.md) | 基于场景上下文的事件触发匹配 |
| [口上系统](口上系统.md) | 事件/指令 key 到对话文本的条件选择 |
| [关系系统](关系系统.md) | 好感/信赖到关系阶段的推算和应用 |
| [地图与导航系统](地图与导航系统.md) | 港区地点定义、连接关系和玩家移动 |
| [日程系统](日程系统.md) | NPC 时段位置刷新和遭遇状态管理 |
| [同行与约会系统](同行与约会系统.md) | 同行/约会状态管理和移动同步 |
| [场景上下文系统](场景上下文系统.md) | 世界状态快照，连接状态和条件匹配的桥梁 |

## 系统依赖关系

```
指令系统
  ├── 场景上下文系统（构建 SceneContext）
  ├── 结算系统（SOURCE → 属性变化）
  │   └── 关系系统（结算后更新阶段）
  ├── 事件系统（匹配触发事件）
  │   └── 关系系统（阶段 rank 比较）
  ├── 口上系统（选择对话文本）
  │   └── 事件系统（事件 key 优先于指令 key）
  ├── 标记系统（施加标记）
  ├── 同行与约会系统（执行操作）
  └── 地图与导航系统（地点标签匹配）

地图与导航系统
  └── 同行与约会系统（同行角色跟随移动）

日程系统
  └── 同行与约会系统（同行/约会角色跳过刷新）
```
