# 场景上下文系统

## 概述

场景上下文（SceneContext）是连接世界状态和条件匹配系统的桥梁。它从当前世界状态和角色状态中提取一次快照，供事件系统和口上系统使用，避免条件匹配时直接读取可变的世界状态。

## 数据结构

### SceneContext

| 字段 | 类型 | 来源 |
|------|------|------|
| `actor_key` | str | actor.key |
| `actor_tags` | [str] | actor.tags |
| `action_key` | str | 当前指令 key 或事件 key |
| `current_day` | int | world.current_day |
| `time_slot` | str | world.current_time_slot.value |
| `location_key` | str | world.active_location.key |
| `location_tags` | [str] | 地点定义的 tags |
| `affection` | int | actor.affection |
| `trust` | int | actor.trust |
| `obedience` | int | actor.obedience |
| `relationship_stage` | str | actor.relationship_stage.key（无则 "stranger"） |
| `relationship_rank` | int | actor.relationship_stage.rank（无则 0） |
| `is_following` | bool | actor.is_following |
| `is_on_date` | bool | actor.is_on_date |
| `is_same_room` | bool | actor.is_same_room |
| `visible_count` | int | 当前地点角色数 |
| `is_private` | bool | visible_count ≤ 1 |
| `marks` | {str: int} | dict(actor.marks) 的副本 |

**不可变性**：SceneContext 是 `frozen=True` 的 dataclass，创建后不可修改。

## 流程

### 构建时机

指令执行过程中构建 3 次 SceneContext：

```
1. 执行前（scene）— 记录指令执行时的原始场景
2. 结算后、操作前（trigger_scene）— 用于事件触发
3. 操作和标记后（settled_scene）— 用于口上选择和事件回退匹配
```

多次构建的原因：结算会改变 affection/trust 等值，操作会改变 is_following/is_on_date，标记会改变 marks。不同阶段需要不同快照。

### 条件匹配中的使用

```
EventService.triggered_events(scene) — 使用 scene 的所有字段做条件过滤
DialogueService.lines_for(scene, events) — 使用 scene 做口上条件匹配
```

## 扩展方法

1. **加新上下文字段**：在 SceneContext 增加字段，在 SceneService.build_for_actor() 中填充
2. **加新匹配条件**：在 EventService/DialogueService 的条件检查逻辑中增加对应字段的判断
