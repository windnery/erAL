# 口上系统

## 概述

口上系统根据事件 key 或指令 key 选择对话文本。选择逻辑：优先匹配事件 key，无匹配则回退到指令 key。同一 key 下可能有多个条目（不同条件/优先级），通过条件过滤和优先级排序选出最佳匹配。

## 数据结构

### DialogueEntry

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 匹配键（事件 key 或指令 key） |
| `actor_key` | str | 角色 key，`"_any"` 表示全局回退 |
| `lines` | [str] | 对话文本行 |
| `required_stage` | str? | 最低关系阶段 |
| `time_slots` | [str] | 时段过滤 |
| `location_keys` | [str] | 地点过滤 |
| `min_affection` | int? | 最低好感 |
| `min_trust` | int? | 最低信赖 |
| `min_obedience` | int? | 最低服从 |
| `requires_private` | bool? | 是否需要私密场景 |
| `requires_date` | bool? | 是否需要约会 |
| `requires_following` | bool? | 是否需要同行 |
| `required_marks` | {str: int} | 需要的标记 |
| `priority` | int | 优先级，默认 0 |

## 流程

### 口上选择

```
1. DialogueService.lines_for(scene, triggered_events)
2. 遍历 triggered_events，对每个 event_key 调用 _lookup()
3. 第一个返回非空结果的 event_key 即为选中口上
4. 若事件口上全部为空，回退到 scene.action_key 查找
```

### _lookup 选择逻辑

```
对指定 key 的所有条目：
1. key 不匹配 → 跳过
2. actor_key == "_any"：
   - 条件不满足 → 跳过
   - 更新 fallback（保留最高优先级）
3. actor_key == scene.actor_key：
   - 条件不满足 → 跳过
   - 更新 best（保留最高优先级）
4. 返回 best.lines，若无则返回 fallback.lines，再无则返回空元组
```

**优先级规则**：角色专属条目 > `_any` 全局回退。同级别内高优先级 > 低优先级。

### 条件匹配（_matches）

所有条件必须同时满足（AND 逻辑）：

- required_stage 匹配
- time_slots 包含当前时段
- location_keys 包含当前地点
- min_affection/trust/obedience 满足
- requires_private/date/following 满足
- required_marks 全部满足

## 配置

口上定义分两层：

- **全局口上**：`data/base/dialogue.toml`——使用 `actor_key = "_any"` 作为回退
- **角色口上**：`data/base/characters/<key>/dialogue.toml`——`actor_key` 必须匹配角色 key

当前全局口上条目（验证用）：

| key | required_marks | lines |
|-----|---------------|-------|
| scold | angry ≥ 1 | 她的表情一下子变得很难看。 |
| tease | embarrassed ≥ 1 | 她的脸一下子红透了。 |
| clink_cups | drunk ≥ 1 | 她喝得双颊微红，眼神有些朦胧。 |

## 扩展方法

1. **加角色口上**：在角色包的 `dialogue.toml` 中加 `[[entries]]`，设置 actor_key 为角色 key
2. **加条件差分**：同一 key 加多条 entry，用不同的 required_stage/required_marks/time_slots 等条件区分，设置不同的 priority
3. **加全局回退口上**：在 `data/base/dialogue.toml` 中用 `actor_key = "_any"` 和 `priority = 1`
4. **加新条件维度**：在 DialogueEntry 增加字段，在 _matches() 增加检查逻辑
