# 资源与物品系统

## 概述

资源与物品系统负责角色持有的消耗品、恢复品、任务物品、礼物与装备槽位。它是道具驱动玩法的基础，也是商店、恢复、事件奖励和关系礼物的承载层。

## 当前状态

- 运行时尚未有完整的背包/物品/装备层。
- 体力系统已经预留了“道具恢复”入口，但还没有统一的物品管理层。

## 建议数据结构

### ItemDefinition

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 物品 key |
| `display_name` | str | 显示名称 |
| `category` | str | 类型：consumable / gift / quest / equip |
| `description` | str | 说明 |
| `stackable` | bool | 是否可堆叠 |
| `max_stack` | int | 最大堆叠数 |
| `price` | int | 价格 |

### InventoryEntry

| 字段 | 类型 | 说明 |
|------|------|------|
| `item_key` | str | 物品 key |
| `count` | int | 数量 |
| `equipped` | bool | 是否装备中 |

### ItemEffect

| 字段 | 类型 | 说明 |
|------|------|------|
| `stamina_restore` | int | 恢复体力 |
| `spirit_restore` | int | 恢复气力 |
| `marks_add` | {str: int} | 施加标记 |
| `source_add` | {str: int} | 作为行动产出的增量 |
| `required_stage` | str? | 使用门槛 |

## 建议流程

1. 获取物品或购买物品时写入库存。
2. 使用物品时先做门槛判断，再消耗数量。
3. 恢复类物品走统一恢复入口，不直接改核心字段。
4. 礼物类物品和关系系统联动。
5. 装备类物品在穿戴后影响恢复、门槛或事件。

## 与现有系统的关系

- 影响 [体力与气力系统](体力与气力系统.md)
- 影响 [关系系统](关系系统.md)
- 影响 [事件系统](事件系统.md)
- 可以作为 [横切支撑系统](横切支撑系统.md) 的外部输入之一
