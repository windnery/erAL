# 系统总览与缺口

## 目的

这份文档分两部分：

1. 汇总 erAL 当前已经实现的系统，并给出粗略完成度。
2. 对比 eraTW，列出当前还缺少、但值得提前规划的系统。

这里的完成度是开发判断，不是验收结论。它反映的是“代码是否已经形成稳定骨架”和“文档是否已经贴合当前实现”。

### 这里的结论

当前 erAL 已经不是“只有几份草稿”的状态，而是已经具备一套可跑的核心玩法闭环：

- 场景快照
- 指令判定与执行
- SOURCE 结算
- 关系推进
- 事件和口上
- 地图与日程
- 同行/约会
- 体力/气力

真正缺的不是主链本身，而是围绕主链的扩展系统和更接近 eraTW 的内容层。

## 二、相比 eraTW 还缺少哪些系统

下面按“建议补文档的优先级”分组。这里不是说全部都要做，而是说这些系统值得提前规划，避免后面内容扩展时没有统一口径。

### P0：建议尽快规划

#### 0. 委托 + 工作 + 双资金 + 港区开发（一组联动）

当前状态：

- 新增了四份规格文档，但运行时尚未实现。
- 这四个系统建议作为一组推进，避免先做收益入口、后补资金出口导致经济失衡。

建议文档入口：

- [委托系统](委托系统.md)
- [工作系统](工作系统.md)
- [资金与账本系统](资金与账本系统.md)
- [港区开发系统](港区开发系统.md)

建议落地顺序：

1. 先做资金与账本（统一记账）。
2. 再做工作（稳定收益）。
3. 再做委托（后台收益 + 角色减载）。
4. 最后做港区开发（资金出口 + 长期目标）。

#### 1. 天气与季节系统

eraTW 参考：`日時天候管理`、`天候管理拡張`、`天候予測システム`

当前状态：

- erAL 里没有独立的天气系统文档，也没有对应的运行时系统。
- `TFLAG` 已经作为临时状态轴存在，但还没有天气层的正式语义。

建议补文档：

- [天气系统](天气系统.md)（未创建时，先作为后续规划入口）

建议包含的内容：

- 当前天气状态
- 天气对地点/指令/恢复的倍率
- 预报与区域天气
- 天气与事件联动

#### 2. 道具 / 背包 / 装备 / 商店系统

- 体力系统里已经预留了“道具恢复”入口，但还没有完整的物品层。

建议补文档：

- [资源与物品系统](资源与物品系统.md)

- 道具对恢复、关系和事件的影响

#### 3. ABL 技能树与训练系统

## 三、建议的后续文档落点

如果你要继续补 specs，建议按下面顺序补，而不是按 eraTW 目录逐项平移：

1. 先补 P0：天气、物品/商店、ABL 成长、PALAM 衰减。
2. 再补 P1：记忆与解锁、关系扩展。
3. 最后再考虑 P2：战斗、小游戏、特殊模式。

这样做的原因很直接：这些系统和你现在的港区日常主循环关系最紧，能最早提升内容生产效率，也最不容易把结构带歪。
