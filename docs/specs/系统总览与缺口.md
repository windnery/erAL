# 系统总览与缺口

## 目的

这份文档分两部分：

1. 汇总 erAL 当前已经实现的系统，并给出粗略完成度。
2. 对比 eraTW，列出当前还缺少、但值得提前规划的系统。

这里的完成度是开发判断，不是验收结论。它反映的是"代码是否已经形成稳定骨架"和"文档是否已经贴合当前实现"。

### 这里的结论

当前 erAL 已具备完整的核心玩法闭环：场景快照 → 指令判定 → SOURCE 结算 → 关系/调教推进 → 事件/口上 → 地图移动 → 时间推进。调教系统主干已完成，经济循环已闭环，地图系统已成熟。

真正缺的不是主链本身，而是围绕主链的深化系统（刻印、反击、ABL 消费）和更接近 eraTW 规模的内容层。

## 二、相比 eraTW 还缺少哪些系统

以下按"对玩法影响程度"分组。

### P0：影响核心玩法深度

#### 1. 调教刻印系统

eraTW 参考：`TRACHECK_刻印取得.ERB`

当前状态：
- erAL 有 mark 基础设施（施加/条件匹配/场景传递），但缺少调教专用的自动刻印逻辑。
- 调教结果态（高潮/拒绝/屈服）已能检测，但没有长期刻印产出。

erAL 不含时姦刻印（无时停系统）。需要的刻印类型：

| 刻印 | 触发条件 | 长期效果 |
|------|---------|---------|
| 快乐刻印 | 连续高潮或高强度快感累积 | 提升对应部位快感 SOURCE 倍率 |
| 屈服刻印 | 反复屈服行为 + submission 阈值 | 降低拒绝概率，解锁主动请求行为 |
| 苦痛刻印 | 高强度 pain 累积 | 影响恐惧/服从 SOURCE |

建议文档入口：[调教刻印系统](调教刻印系统.md)（待创建）

#### 2. ABL 升级的实际消费

eraTW 参考：ABL 等级直接影响指令可用性、SOURCE 倍率、口上分支

当前状态：
- erAL 有 `abl_upgrade.toml` 配置和 `check_and_apply_abl_upgrades` 检查逻辑。
- ABL 升级后没有消费方——没有命令或事件根据 ABL 等级改变行为。
- TALENT 效果已有（敏感度/关系/快感倍率），但 ABL 等级本身不参与门禁或结算。

需要补的内容：
- ABL 等级作为 `required_conditions` 消费（已有管道，缺数据）
- ABL 等级影响 SOURCE 倍率（如 `abl_10` 高 → 服从类 SOURCE 放大）
- ABL 等级影响口上分支条件

建议更新：[ABL 成长系统](ABL成长系统.md)（已有文档，需补充消费层）

#### 3. 反击/主动系统（COUNTER）

eraTW 参考：`COUNTER_SELECT.ERB`、`COUNTER_SOURCE.ERB`、`COUNTER_MESSAGE.ERB`

当前状态：
- erAL 完全缺失这个系统。调教是单向"玩家执行→角色被动接受"。

建议补的内容：
- 调教中角色根据状态（lust/submission/obedience）主动发起行为
- 角色主动行为产出额外 SOURCE（如主动亲吻→追加 pleasure_m）
- 角色主动行为触发专属口上
- 角色可能拒绝、反击或请求特定行为

建议文档入口：[调教反击与主动系统](调教反击与主动系统.md)（待创建）

### P1：影响全局玩法氛围

#### 4. 天气系统

eraTW 参考：`日時天候管理.ERB`、`天候管理拡張.ERB`

当前状态：
- 文档已创建（[天气系统](天气系统.md)），但运行时未实现。
- 季节系统已接入（30天月 + season_month_map），但天气层未实现。

#### 5. PALAM 自然衰减

eraTW 参考：PALAM 的时段变化与状态自然流动

当前状态：
- 文档已创建（[PALAM 衰减系统](PALAM衰减系统.md)），但运行时未实现。
- 当前 PALAM 只增不减（除非高潮清零），需要时段推进时的自然回落。

#### 6. 射精/液体系统

eraTW 参考：`SOURCE_射精確定前処理.ERB`、`SOURCE_射精絶頂表記.ERB`

当前状态：
- erAL 完全缺失。没有玩家侧射精判定、精液量管理、避孕逻辑。
- `lubrication` PALAM 轴和 `fluid` SOURCE 轴已存在但未与液体系统联动。

### P2：内容量与体验

#### 7. 衣服系统深化

eraTW 参考：`衣服/BRA.ERB`、`衣服/CLOTHES.ERB`

当前状态：
- erAL 只有 `removed_slots` 简化模型。足够用但不够细腻。
- 不能区分"掀起"vs"脱掉"，不能穿回，没有独立衣服属性。

#### 8. 角色内容量

- eraTW：100+ 角色，每角色数百条口上
- erAL：3 角色（企业/拉菲/标枪），每角色 30+ 对话

#### 9. 泥酔/醉酒深化

eraTW 参考：`泥酔処理.ERB`、`睡眠深度処理.ERB`

当前状态：
- erAL 只有 `drunk` mark 和基本事件分支。缺少醉酒深度系统和睡眠深度系统。

### 明确不做

- 时停系统（erAL 题材不需要）
- 器具系统化（已在调教设计文档中明确排除）
- 捆绑系统化（同上）
- 妊娠/育儿（长期后再考虑）
- 战斗系统（不影响日常主循环）

## 三、推荐实施顺序

按"对玩法体验影响最大、工作量最小"排序：

1. **调教刻印系统** — 复用 mark 基础设施，改动量小，让调教有长期推进感
2. **ABL 升级消费** — 补数据配置即可，管道已存在
3. **天气系统** — 影响全局玩法氛围，文档已就绪
4. **反击/主动系统** — 提升调教互动深度，改动量中等
5. **PALAM 衰减** — 让状态不再只增不减
6. 角色内容量 — 交给设计员通过编辑器推进
