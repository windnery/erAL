# 系统总览与缺口

## 目的

这份文档分两部分：

1. 汇总 erAL 当前已经实现的系统，并给出粗略完成度。
2. 对比 eraTW，列出当前还缺少、但值得提前规划的系统。

这里的完成度是开发判断，不是验收结论。它反映的是"代码是否已经形成稳定骨架"和"文档是否已经贴合当前实现"。

### 这里的结论

当前 erAL 已完成从基础主链到中深度系统的全量铺设：场景快照 → 指令判定 → SOURCE 结算 → 关系/调教推进 → 事件/口上 → 地图移动 → 时间推进 → 天气/刻印/反击/射精 全部打通。

真正缺的不是主链本身，也不是调教深度，而是**内容量**（角色包/口上文本）与**外围氛围系统**（宴会/日记/醉酒深度/衣服分层）。

## 二、相比 eraTW 还缺少哪些系统

以下按"对玩法影响程度"分组。核心深度与玩法氛围的 P0 项已基本补齐。

### P1：氛围与深度（现阶段值得补）

#### 1. 角色状态详情面板

eraTW 参考：状态查看面板统一展示关系/情感/历史里程碑

当前状态：
- memory 系统已记录里程碑（初吻日/首次约会日/誓约日），但 CLI 没有统一的「查看状态」入口集中呈现。

建议：基于 memory + marks + 关系数据，在 CLI 加一页「角色详情面板」，可持续扩展展示字段。

#### 2. 宴会/聚会系统（ENKAI）

eraTW 参考：`宴会/ENKAI_*` 多角色同场事件

当前状态：
- erAL 完全缺失。只有单角色互动。

明确 **第一版原型不做**，待角色包规模达到 10+ 再立项。

#### 3. 衣服系统深化

当前状态：`removed_slots` 简化模型够用但不够细腻，无法区分"掀起"vs"脱掉"、无法穿回。

#### 4. 泥酔/睡眠深度系统

当前状态：仅 `drunk` mark 与基本事件分支，缺深度层。

### P2：内容量与体验

#### 5. 角色内容量

- eraTW：100+ 角色，每角色数百条口上
- erAL：3 角色（企业/拉菲/标枪），每角色 30+ 对话

内容层工作，交由设计员通过编辑器推进。

#### 6. 节日事件

calendar 已有节日标签，但缺节日触发的专属事件模板。

### 明确不做

- 时停系统（erAL 题材不需要）
- 战斗系统（不影响日常主循环）
- 赌场/小游戏（题材不符）
- 器具系统化（已在调教设计文档中明确排除）
- 捆绑系统化（同上）
- 妊娠/育儿（长期后再考虑）
- 农业/农场（题材不符）

## 三、推荐实施顺序

P0 深度补丁已全部落地（刻印/ABL 消费/反击/天气/PALAM 衰减/射精/随机事件/跨区流动）。下一阶段推荐顺序：

1. **角色状态详情面板** — CLI 层扩展，复用现有数据，低成本高体验
2. **商店 NPC 化** — 明石/不知火作为商店常驻 NPC，打开购物面板的入口从地点挂载改为 NPC 命令
3. **衣服系统深化** — 掀起/穿回语义补齐
4. **泥酔/睡眠深度** — 让持续状态系统更立体
5. 角色内容量 — 内容层工作，由设计员推进
