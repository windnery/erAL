# 结算系统

## 概述

结算系统负责将指令产生的 SOURCE 值转化为实际的属性变化。这是游戏核心数据流的枢纽——所有数值变动都必须经过 SOURCE → 结算 的管道，指令不得直接修改 BASE/PALAM/CFLAG。

## 数据结构

### SOURCE（指令产出）

SOURCE 是指令的"产出声明"，定义在 `commands.toml` 的 `source` 字段中：

```toml
source = { affection = 2, joy = 1 }
```

SOURCE 键名是语义化的（`affection`, `joy`, `trust` 等），不是最终存储位置。

### SettlementRule（结算规则）

每条规则定义一个 SOURCE 键到目标属性家族的映射：

| 字段 | 类型 | 说明 |
|------|------|------|
| `source` | str | SOURCE 键名 |
| `target_family` | str | 目标家族：`base`/`palam`/`cflag`/`tflag` |
| `target_key` | str? | 命名轴目标键名（base/palam 时使用） |
| `target_index` | int? | 兼容轴目标索引（cflag/tflag 时使用） |
| `scale` | int | 倍率，默认 1 |

### AppliedChange（结算结果）

| 字段 | 说明 |
|------|------|
| `family` | 目标家族名 |
| `target` | 目标键名或索引字符串 |
| `before` / `after` | 变化前后值 |
| `delta` | 实际变化量 |

## 流程

```
1. 指令执行 → command.source 写入 actor.stats.source
2. SettlementService.settle_actor() 遍历所有规则
   - 读取 actor.stats.source.get(rule.source)
   - 值为 0 则跳过
   - delta = source_value × rule.scale
   - 写入目标家族（base/palam/cflag/tflag）
   - 记录 AppliedChange
3. actor.sync_derived_fields() — 从 CFLAG 同步 affection/trust/obedience
4. RelationshipService.update_actor() — 重新计算关系阶段
5. actor.stats.clear_source() — 清空 SOURCE 块
```

**关键约束**：SOURCE 必须在结算后清空，否则同一 SOURCE 会重复生效。

## 配置

文件路径：`data/base/settlement_rules.toml`

当前规则覆盖的 SOURCE 键：

| SOURCE 键 | 目标 | 说明 |
|-----------|------|------|
| affection | palam.favor + cflag[2] | 好感 |
| joy | base.mood + cflag[4] | 心情（joy 同时写入 trust 的 cflag[4]） |
| trust | cflag[4] | 信赖 |
| service | cflag[4] | 奉侍（同样写入信赖） |
| obedience | palam.obedience + cflag[6] | 服从 |
| lust | palam.lust | 情欲 |
| temptation | palam.lust | 诱惑（叠加到情欲） |
| submission | palam.submission | 顺从 |
| pain | palam.pain | 痛苦 |
| fear | palam.fear | 恐惧 |
| disgust | palam.disgust | 厌恶 |
| humiliation | palam.shame | 羞辱 |
| unclean | palam.discomfort | 不洁 |
| conquest | palam.superiority | 征服 |
| exposure | palam.shame | 露出 |

## 扩展方法

1. **加新 SOURCE 键**：在 `commands.toml` 的 source 字段中使用新键名，然后在 `settlement_rules.toml` 中加一条映射规则
2. **加新结算目标**：在 `settlement_rules.toml` 中加 `[[rules]]`，指定 source、target_family、target_key/target_index、scale
3. **修改倍率**：调整 scale 值，如 `scale = 2` 表示 SOURCE 值翻倍写入目标
