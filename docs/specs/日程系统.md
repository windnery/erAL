# 日程系统

## 概述

日程系统管理 NPC 在不同时段的位置刷新。每个角色有一个静态日程表（时段 → 地点），时段推进时自动刷新。同行或约会中的角色跳过刷新，跟随玩家移动。

## 数据结构

### CharacterDefinition.schedule

`dict[str, str]`——键为时段名（morning/afternoon/evening 等），值为地点 key。定义在角色包的 `character.toml` 中。

## 流程

### 日程刷新

```
1. 时段推进时调用 ScheduleService.refresh_world(world)
2. 遍历所有角色：
   - is_following 或 is_on_date → 跳过（跟随玩家，不走日程）
   - 查找角色定义中的 schedule[current_time_slot]
   - 有变化 → 更新 location_key，清空 encounter_location_key
   - 记录 previous_location_key
```

### 遭遇状态管理

```
- encounter_location_key 标记角色最后一次被玩家遭遇的地点
- 日程刷新移动角色后，encounter_location_key 不变
- 下次玩家到达新地点时，如果角色在该地点且 encounter_location_key 不同 → 新遭遇
- NavigationService.move_player() 中更新 encounter_location_key
```

## 配置

日程定义在角色包 `data/base/characters/<key>/character.toml` 的 `[schedule]` 段中。

## 扩展方法

1. **修改角色日程**：编辑角色包的 character.toml 的 schedule 段
2. **动态日程**：当前日程是静态的，未来可根据关系阶段、事件等改变角色位置
3. **日程事件**：未来可在日程刷新时触发事件（如角色到达特定地点时的随机事件）
