# 天气系统

## 概述

天气系统管理港区在不同时段、不同区域下的天气状态，并把天气结果投影到移动、恢复、事件和文本条件中。它对应 eraTW 里的日期天气管理链，但实现上应保持港区题材的区域化，而不是照搬原作地名。

## 当前状态

- 运行时尚未实现独立天气系统。
- `TFLAG` 已经存在，但还没有正式的天气语义层。
- 当前只应把天气视为后续规划，不要在指令或事件里假定它已生效。

## 建议数据结构

### WeatherState

| 字段 | 类型 | 说明 |
|------|------|------|
| `weather_key` | str | 当前天气标识 |
| `display_name` | str | 显示名称 |
| `severity` | int | 强度等级 |
| `is_raining` | bool | 是否降雨 |
| `is_storming` | bool | 是否风暴 |
| `area_modifiers` | dict[str, float] | 区域倍率 |

### WeatherDefinition

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 天气 key |
| `display_name` | str | 中文名称 |
| `base_weight` | int | 基础权重 |
| `movement_modifier` | float | 移动修正 |
| `recovery_modifier` | float | 恢复修正 |
| `event_tags` | [str] | 事件标签 |

## 建议流程

1. 世界推进到新时段时刷新天气。
2. 天气写入世界级临时状态。
3. 场景上下文读取天气后，影响指令门控、体力恢复和事件条件。
4. 必要时显示天气预报，作为日程与移动的提示。

## 与现有系统的关系

- 影响 [体力与气力系统](体力与气力系统.md)
- 影响 [地图与导航系统](地图与导航系统.md)
