# ABL 成长系统

## 概述

ABL 成长系统负责能力成长、训练分组、经验转换和技能树扩展。当前 erAL 已经有 ABL 升级骨架，但还缺少“训练体系 + 技能树 + 指令联动”的完整闭环，所以这里单独作为扩展规格。

## 当前状态

- 已有 [横切支撑系统](横切支撑系统.md) 里的 ABL 升级骨架。
- 已能从 `SOURCE` 累积 `abl_*` 经验并推进等级。
- 还没有完整的训练分组、技能树和 ABL 对指令的系统化影响。

## 建议数据结构

### ABLDefinition

| 字段 | 类型 | 说明 |
|------|------|------|
| `abl_index` | int | ABL 索引 |
| `label` | str | 名称 |
| `base_demand` | int | 基础需求 |
| `rate` | float | 成长倍率 |

### TrainingGroup

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 训练组 key |
| `display_name` | str | 显示名称 |
| `source_keys` | [str] | 对应的 SOURCE 项 |
| `affected_abl` | [int] | 受影响的 ABL 索引 |

### ABLNode

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 节点 key |
| `requires` | dict[int, int] | 前置 ABL 等级 |
| `effects` | dict[str, int] | 对指令/恢复/关系的修正 |

## 建议流程

1. 指令或训练行为产出 ABL 经验。
2. 结算后检查是否满足升级阈值。
3. 升级后重新计算相关指令门槛或修正。
4. 技能树节点在满足前置条件后解锁。

## 与现有系统的关系

- 依赖 [结算系统](结算系统.md)
- 依赖 [横切支撑系统](横切支撑系统.md)
- 会影响 [指令系统](指令系统.md) 和 [体力与气力系统](体力与气力系统.md)
