# 记忆与解锁系统

## 概述

记忆与解锁系统负责跨事件、跨时段、跨天数的长期追踪。它记录角色经历过什么、触发过什么、累计过几次，并把这些记录用于后续事件、对话和玩法解锁。

## 当前状态

- 目前没有独立记忆层。
- MARK、CFLAG、TFLAG 已能承载一部分状态，但还不足以覆盖“长期历史追踪”需求。

## 建议数据结构

### MemoryRecord

| 字段 | 类型 | 说明 |
|------|------|------|
| `actor_key` | str | 角色 key |
| `event_key` | str | 事件 key |
| `count` | int | 触发次数 |
| `last_day` | int | 最近触发天数 |
| `flags` | dict[str, int] | 自定义解锁标记 |

### UnlockRule

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 解锁项 key |
| `requires_events` | [str] | 需要出现过的事件 |
| `min_count` | int | 最低累计次数 |
| `required_marks` | {str: int} | 标记门槛 |

## 建议流程

1. 事件触发后记录到记忆层。
2. 进入新时段或新天时更新长期计数。
3. 检查解锁规则，决定是否放开新指令、事件或口上。
