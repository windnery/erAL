# 日常随机事件池（Ambient Events）

## 概述

在时段推进时（`GameLoop.advance_time`）以小概率触发氛围事件，让日常循环不再完全可预测。事件只携带文本与标签，**不直接改变数值**——目的是给"等下一时段"与"路过某地"增加变数。

与指令触发的 `EventService` 的区别：

| 维度 | EventService | AmbientEventService |
|------|-------------|-----|
| 触发 | 玩家执行指令时 | 时段推进时（非玩家动作） |
| 匹配 | action_key + 场景条件 | 场景条件 + 权重 |
| 数值 | 可追加文本/tag，并不直接改 | 纯氛围，不改数值 |
| 频次 | 每次指令都尝试匹配 | 每时段一次 dice，冷却天数内不重复 |

## 数据结构

`data/base/ambient_events.toml`：

```toml
[[events]]
key = "evening_bell"
weight = 10                     # 加权选择
time_slots = ["evening"]         # 时段过滤
location_tags = ["public"]       # 当前位置 tag 过滤（任一命中即可）
seasons = []                     # 季节过滤（空=任意）
weather_keys = []                # 天气过滤（空=任意）
cooldown_days = 3                # 同事件最短间隔
message = "港区的钟声响起..."
tags = ["ambient_evening"]
```

## 运行流程

```
GameLoop.advance_time()
  → ambient_event_service.roll(world)
     → rng.random() >= trigger_chance?   返回 None
     → 筛选 eligible: time_slot / season / weather / location_tag 全匹配
     → 过滤 cooldown_days 内刚触发过的
     → weighted random pick
     → 记录 world.conditions["ambient_last_<key>"] = current_day
     → 返回 AmbientEventOutcome
  → 若命中：event_bus.publish("ambient_event.fired", key/message/tags)
  → runtime_logger.append(triggered_events=[key])
```

## 配置参数（服务字段）

| 字段 | 默认 | 说明 |
|------|------|------|
| `trigger_chance` | 0.25 | 每次 advance_time 的触发总概率 |
| `rng` | `random.Random()` | 注入随机源，便于测试 |

## 冷却机制

冷却键 `ambient_last_<event_key>` 存在 `world.conditions` 中。距离 `world.current_day` 不足 `cooldown_days` 的事件自动排除。冷却数据随存档一起 roundtrip。

## 扩展建议

1. 按权重扩容事件池（每区域 3–5 条起步）
2. 给事件挂 `dialogue_key` 后，让 DialogueService 匹配更具情景感的文本
3. 后续接入 festival_tags / weather_key 做节日与天气专属事件
