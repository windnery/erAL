# 委托系统

## 概述

委托系统用于派遣舰娘在后台执行任务，按时返回资源。它同时承担两件事：

1. 给主循环提供稳定资源来源。
2. 降低当前时段活跃角色数量，减轻运行时压力。

## 当前状态

- 目前未实现独立委托系统。
- 已有日程、地图、同行与约会系统，可作为委托状态切换的基础。

## 数据结构

### CommissionDefinition

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 委托 key |
| `display_name` | str | 显示名称 |
| `duration_slots` | int | 持续时段数 |
| `min_stage` | str? | 最低关系阶段要求 |
| `personal_income` | int | 个人资金奖励 |
| `port_income` | int | 港区资金奖励 |
| `reward_items` | {str: int} | 物品奖励 |

### CommissionAssignment

| 字段 | 类型 | 说明 |
|------|------|------|
| `commission_key` | str | 委托 key |
| `actor_key` | str | 角色 key |
| `start_day` | int | 开始天数 |
| `start_slot` | str | 开始时段 |
| `remaining_slots` | int | 剩余时段 |
| `status` | str | running/success/failed |

## 流程

1. 玩家在可用时段选择委托并指定角色。
2. 角色进入 `on_commission` 状态，不再参与当前场景可见列表。
3. 每次推进时段时减少 `remaining_slots`。
4. 归零后结算奖励，角色回到常规日程流。
5. 在结算日志中记录委托结果。

## 约束

- 同一角色不能同时处于：`is_on_commission`、`is_following`、`is_on_date`。
- 委托中角色不可被普通互动指令选择。
- 委托应以稳定收益为主，不引入复杂编队和战斗模拟。

## 与现有系统关系

- 依赖 [日程系统](日程系统.md) 的时段推进。
- 依赖 [资金与账本系统](资金与账本系统.md) 发放奖励。
- 可与 [资源与物品系统](资源与物品系统.md) 结算物品奖励。

## 字段对齐说明

- 角色状态命名建议与 `CharacterState` 现有风格保持一致，使用 `is_on_commission`（布尔）。
- 委托时段与世界时段保持同一枚举语义，使用现有 `TimeSlot` 值。
- 若需要回写存档，需同步扩展 `SaveService` 的 actor 序列化字段。
