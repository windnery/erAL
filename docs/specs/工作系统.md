# 工作系统

## 概述

工作系统是玩家主动换取资源的稳定入口，定位为低复杂度、可预期收益的日常行为。

## 当前状态

- 目前没有独立工作系统。
- 已有指令系统和体力系统，可直接承接工作指令。

## 数据结构

### WorkActionDefinition

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 工作指令 key |
| `display_name` | str | 显示名称 |
| `time_slots` | [str] | 可执行时段 |
| `downbase` | {str: int} | 与现有指令一致的消耗结构（stamina/spirit） |
| `personal_income` | int | 个人资金收益 |
| `port_income` | int | 港区资金收益 |

## 流程

1. 指令可用性检查（时段、体力、气力）。
2. 扣除 `downbase`（stamina/spirit）。
3. 增加个人资金或港区资金。
4. 输出简单工作结果文案。

## 建议指令

- `office_shift`：标准工作，收益稳定。
- `extra_shift`：额外工作，消耗更高，收益略高。

## 与现有系统关系

- 接入 [指令系统](指令系统.md)。
- 消耗走 [体力与气力系统](体力与气力系统.md)。
- 收益记账走 [资金与账本系统](资金与账本系统.md)。

## 字段对齐说明

- 工作动作建议直接复用现有 `CommandDefinition`，避免再造一套并行结构。
- 消耗字段统一使用现有 `downbase`，键名沿用 `stamina` / `spirit`。
- 时段仍然使用当前项目已有时段字符串：`dawn/morning/afternoon/evening/night/late_night`。
