# 射精与液体系统

## 概述

追踪玩家在插入状态下的兴奋积累，达到阈值后触发射精事件，并根据内外射切换对角色 `fluid` SOURCE 产生影响。补完性交互闭环，连接原先孤立的 `lubrication` PALAM 与 `fluid` SOURCE 两条轴。

## 数据结构

**世界状态**（存 `world.conditions`，随存档 roundtrip）：

| key | 默认 | 说明 |
|-----|------|------|
| `player_arousal` | 0 | 兴奋累积值，0 ~ threshold×2 之间 |
| `ejaculate_inside` | 1 | 内射=1 / 外射=0 |

**服务常量**：

| 常量 | 默认 | 说明 |
|------|------|------|
| `AROUSAL_THRESHOLD` | 100 | 触发射精的下限 |
| `DEFAULT_AROUSAL_GAIN` | 20 | 每次插入指令的累积量 |
| `INSERTED_STATES` | `{"inserted_v", "inserted_a"}` | 哪些持续状态算"插入" |

## 触发流程

```
CommandService.execute(command)
  → settle_actor
  → training_settlement (detect_results + counter + orgasm)
  → if command.requires_training AND ejaculation_service:
       accumulate(world, actor, DEFAULT_AROUSAL_GAIN)
         if not inserted: skip
         else: arousal += gain (clamp to threshold * 2)
       check_and_fire(world, actor)
         if arousal < threshold: return None
         inside = get_inside(world)
         reset_arousal
         if inside:
            actor.stats.source.add("fluid", 30)
            actor.stats.source.add("pleasure_v", 10)
            return "player_ejaculation_inside"
         else:
            return "player_ejaculation_outside"
  → ejaculation_tag 并入 training_tags → dialogue_keys
```

## 内外切换

新增指令 `toggle_ejaculate_inside`（category=training，location=private，time=evening/night/late_night）。
执行后 `ejaculation_service.toggle_inside(world)` 反转 `ejaculate_inside`。

## 对话挂接

两个 fallback tag 在 `dialogue.toml` 可直接匹配：

- `player_ejaculation_inside`
- `player_ejaculation_outside`

后续补专属口上时按角色 + state 差分编辑。

## 与其他系统的关系

- **持续状态**：依赖 `inserted_v` / `inserted_a` 作为触发门禁
- **训练结算**：射精 tag 会并入 `training_tags`，和 counter/orgasm 一起走 dialogue 管线
- **settlement_rules**：`fluid` SOURCE → `lubrication` PALAM 的结算规则已存在，射精会间接提升角色润滑程度
- **记忆系统**：`player_ejaculation_*` 可作为 memory 键被记录（通过 triggered_events 管线）
