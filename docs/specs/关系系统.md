# 关系系统

## 概述

关系系统根据好感（affection）和信赖（trust）计算角色当前的关系阶段。阶段影响指令可用性、事件触发和口上选择。阶段不是直接设置的，而是由 CFLAG 中的好感/信赖值推算得出。

## 数据结构

### RelationshipStage（运行时阶段）

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 阶段标识：stranger/friendly/like/love/oath |
| `display_name` | str | 显示名称 |
| `rank` | int | 阶段序号（0 起），用于比较高低 |

### RelationshipStageDefinition（静态定义）

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 阶段标识 |
| `display_name` | str | 显示名称 |
| `min_affection` | int | 最低好感 |
| `min_trust` | int | 最低信赖 |

## 流程

### 阶段推算

```
1. 遍历所有阶段定义（按 min_affection + min_trust 升序排列）
2. 如果当前 affection ≥ stage.min_affection 且 trust ≥ stage.min_trust → 记录为候选
3. 最终选最后一个满足条件的阶段（最高匹配）
4. 如果没有任何阶段满足 → stranger（rank=0）
```

**语义**：阶段只升不降——即使好感/信赖因特殊原因降低，当前代码不会回退阶段。未来可能需要处理阶段回退。

### 阶段更新时机

```
1. SettlementService.settle_actor() 结算后自动调用 RelationshipService.update_actor()
2. 测试中手动调用 RelationshipService.update_actor(actor)
```

### 阶段在条件匹配中的使用

- **指令**：`required_stage` → 比较 rank
- **事件**：`required_stage` → 比较 rank
- **口上**：`required_stage` → 精确匹配 stage key（不是 rank 比较）

> **注意**：指令和事件用 rank 比较（≥），口上用 key 精确匹配（==）。这是历史遗留的不一致，未来应统一。

## 配置

文件路径：`data/base/relationship_stages.toml`

| 阶段 | display_name | min_affection | min_trust |
|------|-------------|---------------|-----------|
| stranger | 陌生 | 0 | 0 |
| friendly | 友好 | 1 | 0 |
| like | 喜欢 | 3 | 2 |
| love | 爱 | 6 | 4 |
| oath | 誓约 | 10 | 6 |

## 扩展方法

1. **加新阶段**：在 `relationship_stages.toml` 中加 `[[stages]]`，注意 min_affection/min_trust 必须递增
2. **调整阶段门槛**：修改对应阶段的 min_affection/min_trust
3. **加阶段特效**：目前阶段仅影响条件匹配，未来可在阶段变化时触发事件或口上
