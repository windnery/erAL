# 同行与约会系统

## 概述

同行和约会是两个递进的关系状态：先同行（follow），再在同行基础上开启约会（date）。两者都影响角色的移动行为和指令可用性。约会依赖同行——邀请约会会自动开启同行。

## 数据结构

### 角色状态字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `is_following` | bool | 是否正在同行 |
| `follow_ready` | bool | 同行就绪标志 |
| `is_on_date` | bool | 是否正在约会 |

### 兼容轴 CFLAG 映射

> 运行时代码现通过 `src/eral/domain/compat_semantics.py` 中的英文 key 访问这些索引，避免继续散落裸数字；era 兼容索引本身保持不变。

| CFLAG 索引 | 英文 key | 含义 |
|------------|---------|------|
| 12 | on_date | 约会中 |
| 319 | same_room | 同室 |
| 320 | following | 同行 |
| 329 | follow_ready | 同行准备 |

### 世界状态

| 字段 | 说明 |
|------|------|
| `world.date_partner_key` | 当前约会对象 key |
| `world.is_date_traveling` | 约会途中标志（禁用 date 指令） |

## 流程

### 同行

```
start_follow:
1. actor.follow_ready = True
2. actor.is_following = True
3. actor.encounter_location_key = 当前地点（避免重复遭遇）
4. CompanionService.refresh_world() — 通过 compat 语义访问层更新 CFLAG 和 same_room

stop_follow:
1. actor.follow_ready = False
2. actor.is_following = False
3. actor.is_on_date = False — 停止同行同时结束约会
4. CompanionService.refresh_world()
```

### 约会

```
start_date:
1. 如果未同行 → 先 start_follow
2. actor.is_on_date = True
3. DateService.refresh_world() — 通过 compat 语义访问层更新 on_date / date_partner_key
4. CompanionService.refresh_world() — 同步 CFLAG

end_date:
1. actor.is_on_date = False
2. DateService.refresh_world()
3. CompanionService.refresh_world()
4. CommandService 额外以 `after_date_event` 为 action key 构造后事件场景
5. 后事件层单独匹配 `after_date_event` 事件与对话，再附加到 `end_date` 结果
```

### 同行角色移动

```
NavigationService.move_player() 时：
1. 遍历所有 is_following 角色
2. 更新 location_key 为目标地点
3. 更新 encounter_location_key
4. CompanionService.refresh_world() — 同步 same_room 和 CFLAG

当前已接入的语义 key：`same_room / following / follow_ready / on_date`
```

### 日程与同行的交互

```
ScheduleService.refresh_world() 时：
- is_following 或 is_on_date 的角色跳过日程刷新
- 确保同行角色不会因日程而离开玩家
```

## 配置

同行和约会通过指令的 `operation` 字段触发：

| 指令 | operation | 前置条件 |
|------|-----------|---------|
| invite_follow | start_follow | requires_following=false, required_stage=friendly |
| dismiss_follow | stop_follow | requires_following=true |
| invite_date | start_date | requires_following=true, requires_date=false, required_stage=like |
| end_date | end_date | requires_date=true |

## 扩展方法

1. **加新同行/约会指令**：在 `commands.toml` 中定义，设置对应的 operation 和 requires_following/requires_date
2. **同行/约会特殊效果**：目前同行和约会只影响移动和指令可见性，未来可加心情/好感被动加成
3. **约会旅行**：`world.is_date_traveling` 标志已预留，未来实现约会移动时禁用普通 date 指令
4. **约会后事件**：如告白后余韵、约会收尾分支，不再混入普通 `end_date` 事件，而是放到 `after_date_event` 层
