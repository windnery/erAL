# 调教反击与主动系统

## 概述

反击/主动系统让角色在调教过程中不再是纯粹的被动接受者，而是根据自身状态主动发起行为。这对应 eraTW 的 COUNTER 系统。

设计原则：
- 角色主动行为不是独立的命令系统，而是调教结算后的附加 SOURCE + 口上产出
- 角色的主动性与 lust / submission / obedience / 关系阶段挂钩
- 第一版只做最基础的主动行为（主动亲吻、主动爱抚、主动请求），不做完整的 COUNTER 树

## 触发机制

### 触发时机

在 `TrainingService.detect_results()` 之后，新增一步主动行为检测：

```
调教命令执行
  → SOURCE 写入
  → SettlementService.settle_actor()
  → TrainingService.detect_results()
  → TrainingService.detect_counter_action()   ← 新增
     → 判断角色是否主动发起行为
     → 产出 counter SOURCE
     → 产出 counter 口上标签
```

### 触发条件

角色主动行为概率由以下因素决定：

| 因素 | 影响 |
|------|------|
| lust PALAM | 越高越可能主动发起亲密行为 |
| submission PALAM | 越高越可能主动发起奉仕行为 |
| obedience PALAM | 越高越可能配合而非拒绝 |
| 关系阶段 | 喜欢以上更可能主动 |
| 快乐刻印等级 | 刻印越高越可能主动请求 |
| 屈服刻印等级 | 屈服刻印 3 级解锁主动请求 |

### 触发概率公式（建议）

```python
base_chance = 0.0
if lust >= 2000:
    base_chance += 0.15
if lust >= 5000:
    base_chance += 0.15
if submission >= 3000:
    base_chance += 0.1
if obedience >= 2000:
    base_chance += 0.1
if relationship_rank >= 3:  # 喜欢
    base_chance += 0.1
if actor.has_mark("mark_pleasure", 2):
    base_chance += 0.1
if actor.has_mark("mark_submission", 3):
    base_chance += 0.2
final_chance = min(base_chance, 0.8)
```

## 主动行为类型

### 第一版行为

| 行为 | 条件 | 效果 | 口上标签 |
|------|------|------|---------|
| 主动亲吻 | lust ≥ 2000，关系 ≥ 喜欢 | 追加 pleasure_m SOURCE | `counter_kiss` |
| 主动爱抚 | lust ≥ 5000 | 追加 pleasure_c SOURCE | `counter_touch` |
| 主动请求 | 屈服刻印 ≥ 3 或 submission ≥ 10000 | 追加 sexual_act SOURCE | `counter_request` |
| 主动奉仕 | obedience ≥ 4000，submission ≥ 5000 | 追加 give_pleasure_c SOURCE | `counter_service` |

### 后续扩展

- 主动脱衣（counter_undress）
- 主动切换体位（counter_change_position）
- 主动拒绝并反击（counter_resist）
- 根据角色个性差分（不同角色的主动行为偏好不同）

## 数据结构

```python
@dataclass(slots=True)
class CounterAction:
    action_key: str           # counter_kiss / counter_touch / counter_request / counter_service
    source: dict[str, int]    # 追加 SOURCE
    dialogue_key: str         # 口上标签
```

`TrainingService.detect_counter_action()` 返回 `CounterAction | None`。

## 与现有系统的关系

- **调教结算**：在 `detect_results()` 之后调用，追加 SOURCE
- **口上系统**：counter 标签注入 `triggered_events` 管线，匹配专属口上
- **刻印系统**：屈服刻印 3 级解锁"主动请求"
- **素质效果**：TALENT（如 devoted）可提高主动概率

## 实施范围

5. 测试
