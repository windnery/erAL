# 横切支撑系统

## 概述

横切支撑系统不是一个单独的玩家玩法循环，而是为指令、结算、关系和状态变化提供共用能力的集合。它把容易散落在各处的规则集中起来，避免文档只写主链、忽略实际运行时已经存在的门控和修正层。

当前纳入这一层的能力主要有：

- 指令可用性门控
- 体力/气力处理
- TALENT 造成的 SOURCE / 恢复修正
- ABL 经验与升级
- 刻印/印记阈值触发

## 当前模块

### 1. 指令门控

对应实现：

- [src/eral/systems/command_gates.py](../src/eral/systems/command_gates.py)
- [src/eral/systems/commands.py](../src/eral/systems/commands.py)

职责：

- 检查类别是否启用
- 检查全局模式（忙碌、约会途中）
- 检查地点、时段、关系、同行、约会和标记条件
- 检查体力/气力是否允许执行

### 2. 体力与气力

对应实现：

- [src/eral/systems/vital.py](../src/eral/systems/vital.py)
- [src/eral/systems/game_loop.py](../src/eral/systems/game_loop.py)
- [src/eral/systems/commands.py](../src/eral/systems/commands.py)

职责：

- 处理 `downbase` 直接扣减
- 处理自然恢复、睡眠恢复、小憩恢复、泡澡恢复
- 管理疲劳值和晕倒判定

### 3. TALENT / SOURCE 修正

对应实现：

- [src/eral/systems/source_extra.py](../src/eral/systems/source_extra.py)
- [src/eral/systems/favor_calc.py](../src/eral/systems/favor_calc.py)

职责：

- 在结算前对 `SOURCE` 做天赋修正
- 在关系或成长计算中读取 `TALENT`
- 把“隐藏素质影响数值”的规则集中放在修正层，而不是散在指令里

### 4. ABL 升级

对应实现：

- [src/eral/systems/abl_upgrade.py](../src/eral/systems/abl_upgrade.py)
- [src/eral/content/abl_upgrade.py](../src/eral/content/abl_upgrade.py)

职责：

- 累积 `abl_*` 经验
- 按阈值推进 ABL 等级
- 作为技能树和能力成长的基础

### 5. 刻印

对应实现：

- [src/eral/systems/imprint.py](../src/eral/systems/imprint.py)
- [src/eral/content/imprint.py](../src/eral/content/imprint.py)

职责：

- 根据累计值判断标记是否升级
- 把持续互动的结果沉淀成可匹配的状态标签

## 边界

这一层不负责：

- 指令文本本身
- 地图移动逻辑
- 场景快照构建
- 事件和口上的最终选择

它只负责把“可用性、消耗、成长、阈值、修正”这些共用规则集中管理。

## 规划建议

如果后续继续扩展，优先在这一层下再拆三类文档：

1. 命令门控系统：只讲门槛判定和失败原因。
2. ABL 与天赋系统：只讲技能树、经验和修正。
3. 刻印与状态沉淀系统：只讲持久化的状态升级。
