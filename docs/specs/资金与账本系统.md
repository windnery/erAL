# 资金与账本系统

## 概述

资金系统采用双账户：个人资金与港区资金。账本系统用于记录收入和支出，便于追踪经济流向。

## 当前状态

- 当前未实现统一的双账户和流水。
- 委托、工作、消费类功能需要先依赖此系统。

## 数据结构

### WalletState

| 字段 | 类型 | 说明 |
|------|------|------|
| `personal_funds` | int | 个人资金 |
| `port_funds` | int | 港区资金 |

### LedgerEntry

| 字段 | 类型 | 说明 |
|------|------|------|
| `day` | int | 天数 |
| `time_slot` | str | 时段 |
| `account` | str | personal/port |
| `delta` | int | 变动值 |
| `reason` | str | 来源说明 |
| `source_key` | str? | 关联指令/委托 key |

## 规则

- 个人资金：用于礼物、约会开销、个人消费。
- 港区资金：用于港区设施建设与升级。
- 允许个人资金捐赠到港区资金。
- 不允许港区资金直接转为个人资金。

## 流程

1. 所有加钱/扣钱动作都经过资金服务。
2. 写入对应账户余额。
3. 同步写入账本流水。
4. UI 层显示余额和最近流水。

## 与现有系统关系

- 接入 [委托系统](委托系统.md) 与 [工作系统](工作系统.md) 的收益结算。
- 接入 [资源与物品系统](资源与物品系统.md) 的消费结算。
- 接入 [港区开发系统](港区开发系统.md) 的建设支出。

## 字段对齐说明

- 当前 `WorldState` 尚无资金字段，建议新增到世界层：`personal_funds`、`port_funds`。
- 资金流水建议挂在世界层或独立服务中，字段使用 snake_case。
- 若先做最小版本，可只存余额，不立即落全量流水；后续再补 `LedgerEntry`。
