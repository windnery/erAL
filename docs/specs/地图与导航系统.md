# 地图与导航系统

## 概述

地图系统定义港区地点和连接关系，导航系统管理玩家在地点间的移动。地点有可见性控制——私有地点需要同行/约会状态才能看到和进入。角色移动会触发遭遇检测。

## 数据结构

### PortMapLocation

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | str | 唯一标识 |
| `display_name` | str | 显示名称 |
| `zone` | str | 区域分类 |
| `tags` | [str] | 功能标签（影响指令可用性） |
| `start` | bool | 是否为起始地点 |
| `visibility` | str | 可见性：public/private/hidden |

### PortConnection

| 字段 | 类型 | 说明 |
|------|------|------|
| `source` | str | 起点地点 key |
| `target` | str | 终点地点 key |
| `bidirectional` | bool | 是否双向，默认 true |

### PortMap

地图整体，包含 locations 和 connections。提供：
- `location_by_key(key)` — 按 key 查找地点
- `starting_location()` — 查找起始地点
- `neighbors_of(key)` — 获取所有相邻地点（忽略可见性）
- `visible_neighbors(key, can_see_private)` — 获取可见相邻地点

## 流程

### 可见性规则

```
- public：始终可见
- private：仅当 can_see_private=True 时可见（玩家正在同行或约会）
- hidden：永远不可见（预留）
```

### 移动流程

```
1. 检查目标是否为当前地点 → 已在目的地
2. 检查目标是否在 visible_destinations 中
   - 若在 neighbors_of 中但不在 visible → "目前无法前往X"
   - 不在 neighbors_of → 抛出异常
3. 更新 world.active_location
4. CompanionService.move_followers() — 同行角色跟随移动
5. 遭遇检测：
   - 遍历非同行角色
   - 在目标地点且 encounter_location_key != 目标 → 新遭遇
   - 更新 encounter_location_key
6. 发布 encounter 事件到 EventBus
7. 返回 ActionResult（含移动消息和遭遇消息）
```

### 私有地点访问

```
can_see_private = 任何角色 is_following 或 is_on_date
→ visible_destinations 包含相邻的 private 地点
→ 可移动到私有地点
```

## 配置

文件路径：`data/base/port_map.toml`

地点标签影响指令可用性——指令的 `location_tags` 与地点的 `tags` 做 OR 匹配。

## 扩展方法

1. **加新地点**：在 `port_map.toml` 中加 location 和 connection
2. **加私有地点**：设置 `visibility = "private"`，同行/约会时自动解锁
3. **加隐藏地点**：设置 `visibility = "hidden"`，需在代码中手动解锁逻辑
4. **加新标签**：在地点 tags 中使用新标签，然后在指令 `location_tags` 中引用
