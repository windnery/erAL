# PALAM 衰减系统

## 概述

PALAM 衰减系统负责把角色的累积状态做时间上的回落、自然变化和日程联动。它适合承载欲望、顺从、痛苦、羞耻等需要随时间变化的状态，而不是让这些值只增不减。

## 当前状态

- 结算链已经能写入 PALAM。
- 还没有独立的 PALAM 衰减周期和规则文档。

## 建议数据结构

### PalamDecayRule

| 字段 | 类型 | 说明 |
|------|------|------|
| `palam_key` | str | PALAM 键 |
| `base_decay` | int | 基础衰减 |
| `time_slot_decay` | dict[str, int] | 时段附加衰减 |
| `location_modifier` | float | 地点倍率 |
| `recovering_modifier` | float | 休憩/睡眠倍率 |

## 建议流程

1. 在时段推进或日终时结算衰减。
2. 根据状态、地点和恢复行为调整衰减幅度。
3. 把回落后的 PALAM 再反馈给事件或口上。

## 与现有系统的关系
