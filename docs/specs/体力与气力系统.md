# 体力与气力系统

## 概述

体力与气力是 erAL 中的全局行动资源，用来限制角色在单回合、单时段和单场景内能够承担的行动量。

参考 eraTW 的设计，它们不是只给亲密指令用的资源，而是覆盖以下场景：

- 指令执行消耗
- 休憩、睡眠、睡眠药等恢复流程
- 道具、家具、天赋对恢复效率的修正
- 行动中断、疲劳返家、无法继续行动等状态门槛
- 日常互动、约会、工作、事件结算中的额外消耗

核心原则：

1. 体力表示身体承受力，适合刻画长时、重度、强体位、强刺激行为。
2. 气力表示精神/集中/行动意愿，适合刻画大多数行动的通用消耗。
3. 轻动作主要消耗气力；重动作同时消耗体力和气力。
4. 气力归零时，玩家不能再选择任何会消耗气力的交互。
5. 体力归零时，玩家直接晕倒并推进到下一天；未来接入医务室后，也可以改成送医务室处理。

## 数据结构

### VitalState

| 字段 | 类型 | 说明 |
|------|------|------|
| `current_body` | int | 当前体力 |
| `current_energy` | int | 当前气力 |
| `max_body` | int | 体力上限 |
| `max_energy` | int | 气力上限 |
| `recover_speed` | int | 恢复速度修正，取值建议为 -1/0/1 |
| `fatigue_level` | int | 疲劳等级，用于 UI 和门槛判断 |
| `sleeping` | bool | 是否处于睡眠状态 |
| `resting` | bool | 是否处于休憩状态 |
| `collapsed` | bool | 是否因体力归零而晕倒 |

建议保留当前值与上限值双层结构，避免恢复和扣除逻辑混在同一字段里。

### VitalChange

| 字段 | 类型 | 说明 |
|------|------|------|
| `body_delta` | int | 体力变化量，负数表示消耗，正数表示恢复 |
| `energy_delta` | int | 气力变化量，负数表示消耗，正数表示恢复 |
| `reason` | str | 变化来源，例如 command/rest/item/event |
| `source_key` | str? | 对应指令、道具或事件 key |

### CommandCostProfile

| 字段 | 类型 | 说明 |
|------|------|------|
| `category` | str | 动作分类：social / care / intimacy / hard / special / rest |
| `body_cost` | int | 基础体力消耗 |
| `energy_cost` | int | 基础气力消耗 |
| `intensity` | int | 强度等级，建议 1-5 |
| `requires_body_threshold` | int? | 最低体力门槛 |
| `requires_energy_threshold` | int? | 最低气力门槛 |
| `can_fail_on_fatigue` | bool | 是否会因疲劳直接失败 |
| `energy_costs_action` | bool | 是否属于会消耗气力的交互 |

## 流程

### 1. 行动前判定

```
1. 读取当前体力/气力
2. 计算动作基础消耗
3. 叠加分类倍率、天赋倍率、场景倍率
4. 检查门槛
5. 不满足时返回失败原因或替代行为
```

- 气力为 0 时，所有 `energy_costs_action = true` 的交互都不可选。
- 体力为 0 时，不进入普通失败分支，而是直接触发晕倒流程。

### 2. 行动消耗

总消耗建议采用分层公式：

```
最终消耗 = 基础消耗 × 分类倍率 × 场景倍率 × 天赋倍率 × 状态倍率
```

建议规则：

- `category` 决定基础区间
- `intensity` 决定倍率档位
- 关系状态、是否同行、是否约会、是否工作中，决定附加倍率
- 疲劳越高，后续同类动作的成本可以逐步上升
- 若气力不足，该动作应直接从可选列表中移除，而不是等到执行时报错

### 3. 行动后结算

```
1. 扣除体力/气力
2. 记录疲劳等级变化
3. 若气力降到 0，禁止继续选择所有气力消耗交互
4. 若体力降到 0，触发晕倒流程，强制结束当前行动
5. 按规则推进到下一天，或在未来版本中转入医务室处理
6. 进入日终/场景收尾流程
```

### 4. 恢复流程

恢复不应只靠自然回满，建议拆成四类：

1. **自然恢复**
   - 每个时段结算一次
   - 按最大值百分比恢复
   - 受恢复速度素质影响

2. **休憩恢复**
   - 在休憩地点、座椅、床、房间等场景额外恢复
   - 主要恢复气力，兼顾少量体力

3. **睡眠恢复**
   - 这是最高效的恢复方式
   - 夜间/床铺/同睡/陪睡可进一步提高恢复
   - 睡眠不足时恢复受限，可能叠加疲劳惩罚

4. **道具恢复**
   - 药品、饮食、功能性道具可直接回体力或气力
   - 高恢复道具应伴随副作用、冷却或稀缺性

5. **归零处理**
   - 气力归零只影响“还能不能继续做会消耗气力的动作”，不强制结束当天流程
   - 体力归零直接触发晕倒，当前行动立即中断，并默认推进到下一天

## 配置建议

建议在 `data/base` 中拆成三份配置：

| 文件 | 作用 |
|------|------|
| `vital_params.toml` | 上限、恢复公式、阈值、状态倍率 |
| `vital_costs.toml` | 指令分类、基础消耗、门槛 |
| `vital_recovery.toml` | 休憩、睡眠、道具、家具、天赋修正 |

### 推荐参数起点

| 项目 | 建议值 | 说明 |
|------|------|------|
| 默认体力上限 | 1800-2500 | 普通角色 |
| 默认气力上限 | 1400-2200 | 普通角色 |
| 轻社交消耗 | 0-20 气力 | 会话、打招呼、简单互动 |
| 轻亲密消耗 | 5-50 体力 / 20-100 气力 | 亲吻、抚摸、轻度奉仕 |
| 重亲密消耗 | 40-300 体力 / 80-450 气力 | 长时、重度、双人或特殊姿势 |
| 休憩恢复 | 5%-20% | 以气力为主 |
| 睡眠恢复 | 20%-60% | 以体力和气力双恢复为主 |

建议在指令配置里再补一个布尔字段：

| 字段 | 作用 |
|------|------|
| `energy_costs_action` | 是否占用气力；为 `false` 时，即使气力为 0 也仍可使用 |

这些数值只是 erAL 的起点，不必照搬 eraTW 的绝对值，但应保留“轻动作、重动作、睡眠恢复”三层差异。

## 参考自 eraTW 的可迁移语义

### 1. 角色基础值

eraTW 直接在角色初始表里配置体力和气力，不同角色差异很大。说明这两个资源本身就属于角色成长曲线的一部分，而不是固定常量。

### 2. 天赋影响恢复

eraTW 的“回復速度”会放大或缩小恢复量，日光浴、月光浴、治疗等词条也会影响特定场景下的恢复。

### 3. 道具与家具影响恢复

恢复道具、睡眠药、床铺、沙发等会直接影响恢复结果，说明恢复不是纯系统内置，而是可被内容层强烈干预的。

### 4. 行动不止是性指令

会话、泡茶、身体接触、キス、ピロートーク、何もしない、约会中的送礼和同行类动作，都可能消耗气力，部分还会消耗体力。

### 5. 事件与日终也会扣除

日常事件、特殊玩法和场景后处理也会直接写入体力/气力，说明它们应该被视为“行动预算”，而不是“战斗资源”。

### 6. 归零后的玩家感受

TW 的手感核心不是“两个条都空了都不能动”，而是：

- 气力空了，玩家还可以做不耗气力的事，但不能再继续高频互动。
- 体力空了，游戏应立即停止当前活动，避免玩家硬撑到下一条指令。

## 扩展方法

1. **新增指令消耗**：给指令写 `vital_cost_profile`，再按分类套用倍率。
2. **新增恢复来源**：在休憩、睡眠、道具、家具、天赋里加新的恢复规则。
3. **新增疲劳门槛**：给低体力/低气力配置替代行为或强制失败文案。
4. **新增状态修正**：让同行、约会、睡眠、醉酒、忙碌等状态影响消耗与恢复。
5. **新增 UI 提示**：显示当前体力/气力百分比、疲劳等级和即将失败的原因。
6. **新增归零处理**：气力归零做禁用列表过滤，体力归零做晕倒与日推进。

## 落地顺序建议

1. 先把所有指令补上体力/气力消耗字段。
2. 再做休憩/睡眠/道具恢复。
3. 然后补低体力/低气力门槛。
4. 最后再接入天赋、家具和状态倍率。

这样可以先得到一套可玩、可平衡、可继续加内容的骨架，再逐步向 eraTW 风格靠拢。
