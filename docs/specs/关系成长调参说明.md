# 关系成长调参说明

本文档说明 `relationship_growth.toml` 和 `relationship_stages.toml` 中各参数的含义、调参规则、推荐区间，以及修改后的验证步骤。

---

## 1. 管线总览

```
指令执行 → SOURCE 产出（affection/joy/trust/fear/disgust/humiliation…）
         → SettlementService Phase 1.5
             ├─ FAVOR_CALC → CFLAG:2 (好感度/affection)
             └─ TRUST_CALC → CFLAG:4 (信赖度/trust)
         → RelationshipService 检查阶段迁移
```

**关键公式**（`favor_calc` / `trust_calc` 通用）：

```
net      = Σ(positive_keys) - Σ(negative_keys)
delta    = int(net × base_scale × stage_mult × talent_mult)
delta    = clamp(delta, clamp_min, clamp_max)
delta   *= facility_relation_multiplier  (约会点设施加成)
delta    = max(delta, 0)
```

---

## 2. `relationship_growth.toml` 参数说明

### 2.1 通用参数

| 参数 | 含义 | 当前值 (favor) | 当前值 (trust) | 推荐区间 | 调参效果 |
|------|------|----------------|----------------|----------|----------|
| `base_scale` | 基础缩放系数 | 0.08 | 0.15 | 0.03 ~ 0.20 | 整体增速；越大升级越快 |
| `base_positive_keys` | 正向 SOURCE 求和键 | affection, joy, service, trust | trust, joy, service | — | 增删键改变哪些指令贡献成长 |
| `base_negative_keys` | 负向 SOURCE 求和键 | fear, disgust, humiliation | fear, disgust | — | 增删键改变哪些行为会抑制成长 |
| `clamp_min` | 单次最小值 | 0 | 0 | 0 | 通常不动 |
| `clamp_max` | 单次最大值 | 500 | 300 | 100 ~ 9999 | 防止单次极端增长 |

### 2.2 阶段倍率 `relationship_stage_multiplier`

| 阶段 | favor 倍率 | trust 倍率 | 设计意图 |
|------|-----------|-----------|----------|
| stranger | 0.5 | 0.6 | 初期慢，真实感 |
| friendly | 0.8 | 0.9 | 略有加速 |
| like | 1.0 | 1.0 | 基准速度 |
| love | 1.3 | 1.2 | 高阶段有加成 |
| oath | 1.5 | 1.4 | 誓约后最快 |

**调参规则**：
- 所有值应在 0.3 ~ 2.0 之间
- 必须保持递增：`stranger < friendly < like < love < oath`
- 如果某阶段升级太慢，提高该阶段倍率或降低前一阶段阈值

### 2.3 素质修正 `talent_modifiers`

| era_index | effect | favor 倍率 | trust 倍率 | 含义 |
|-----------|--------|-----------|-----------|------|
| 60 | resistance | -0.1 | -0.1 | 抵抗 |
| 61 | devoted | +0.1 | +0.15 | 献身 |
| 64 | emotionless | -0.2 | -0.15 | 无感情 |
| 78 | yearning | +0.2 | — | 恋慕 |
| 79 | in_love | +0.3 | — | 恋人 |
| 65 | pleasure_response | +0.1 | — | 快感应答 |

**计算逻辑**：`talent_mult = 1.0 + Σ(modifier × sign(talent_value))`，即 TALENT 为正时加、为负时减。

**调参规则**：
- 每条修正量建议在 -0.3 ~ +0.3 之间
- 总累加不应使 `talent_mult` 跌破 0.3（否则几乎不成长）

## 3. `relationship_stages.toml` 参数说明

| 阶段 | key | min_affection | min_trust | min_intimacy | 附加条件 |
|------|-----|--------------|-----------|-------------|----------|
| 陌生 | stranger | 0 | 0 | 0 | — |
| 友好 | friendly | 200 | 100 | 0 | — |
| 喜欢 | like | 400 | 200 | 3 | — |
| 爱 | love | 800 | 400 | 5 | 无反感印记 |
| 誓约 | oath | 2000 | 1000 | 8 | 无反感印记 + 誓约之戒 |

**调参规则**：
- affection/trust 阈值必须严格递增
- intimacy（ABL:9）门槛也必须递增
- 提高阈值 = 升级更慢，降低 = 更快
- 誓约门槛（2000/1000）是长期目标，不宜低于 1500/800

## 4. 速度估算

以当前参数（base_scale favor=0.08）估算日常指令的贡献：

| 指令 | SOURCE (affection) | 阶段 | 单次 favor delta | 升到下一阶约需 |
|------|-------------------|------|-----------------|---------------|
| chat | ~50 | stranger (0.5×) | ~2 | ~100 次 → 友好 |
| chat | ~50 | friendly (0.8×) | ~3 | ~67 次 → 喜欢 |
| touch_head | ~40 | stranger | ~1-2 | 补充 |
| praise | ~30 | stranger | ~1 | 补充 |

> 以上是纯 chat 的估算。实际玩家会混合多种正向指令，且有 date_spot 设施加成和 TALENT 修正，实际速度约为估算的 1.5~2 倍。

**目标节奏**（参考 eraTW）：
- 陌生 → 友好：3~5 天日常互动
- 友好 → 喜欢：5~8 天
- 喜欢 → 爱：8~15 天
- 爱 → 誓约：20~30 天（需亲密度和物品）

---

## 5. 设施加成

`FacilityService.relation_multiplier` 由约会点（date_spot）设施等级决定：

| 等级 | 效果值 | relation_multiplier |
|------|-------|-------------------|
| 0 | 0 | 1.0 |
| 1 | 0.1 | 1.1 |
| 2 | 0.25 | 1.25 |
| 3 | 0.45 | 1.45 |
| 4 | 0.75 | 1.75 |

此加成在 SettlementService Phase 1.5 中应用到 favor_delta 和 trust_delta。

## 6. 调参验证步骤

修改参数后执行以下验证：

```bash
# 1. 全量回归（确保无破坏）
python -m unittest discover -s tests -t .

# 2. 关系相关专项测试
python -m unittest tests.test_relationships tests.test_14day_combined -v

# 3. 内容校验
python -m eral.tools.validate_content --root .
```

**需关注的测试断言**：
- `test_chat_increments_affection`：验证 chat 后 affection 增长值
- `test_14_day_combined_l3_l4_loop`：验证 14 天后 affection ≥ 100
- `test_relationship_stages_ordering_validated`：验证阶段阈值递增

**修改 `base_scale` 后**：更新 `test_chat_increments_affection` 中的期望值（`actor.affection` 和 `palam favor`）。

**修改阶段阈值后**：检查 `tests/support/stages.py` 中 `seed_friendly` / `seed_like` 等辅助函数是否仍能正确越过新阈值（它们从 TOML 动态读取，通常自动适配）。

## 7. 与 eraTW 的对应关系

| erAL 概念 | eraTW 对应 | 备注 |
|-----------|-----------|------|
| affection (CFLAG:2) | 好感度 (CFLAG:2) | 索引一致 |
| trust (CFLAG:4) | 信赖 (CFLAG:4) | 索引一致 |
| favor_calc | eraTW 的 FAVOR_CALC | erAL 简化为 toml 配置 |
| trust_calc | eraTW 的 TRUST_CALC | 同上 |
| relationship_stages | eraTW 的 关系段階 | erAL 为 5 级，eraTW 有更多细分 |
| base_scale | eraTW 的固定系数 | eraTW 用 ERB 硬编码，erAL 提取为参数 |
| talent_modifiers | eraTW 的 TALENT 条件分支 | erAL 简化为倍率叠加 |
