# eraTW COMABLE 规则矩阵

## 1. 目的

这份文档把 `eraTW` 的 `COMABLE` 体系拆成可迁移规则，回答两件事：

1. 指令“能不能显示/能不能执行”是怎么分层判定的。
2. 这些判定怎样映射到 `erAL` 的结构化系统里。

## 4. 总体结构

`eraTW` 的 COMABLE 不是一个单函数，而是两层：

### 4.1 第一层：全局区间判定

主入口：

- `@GLOBAL_COMABLE(ARG)`
- `@GLOBAL_COMABLE2(ARG)`

它根据指令编号区间做快速屏蔽。

### 4.2 第二层：指令专用判定

每条或每组指令有自己的 `@COM_ABLExxx`。

例如：

- `@COM_ABLE86`
- `@COM_ABLE318`
- `@COM_ABLE362`
- `@COM_ABLE611`
- `@COM_ABLE612`

也就是说：

1. 先过全局大类门槛
2. 再过该指令自己的条件门槛

这和 `erAL` 当前只有一层 `available_commands_for_actor()` 不同，后续应拆成两层。

## 5. 全局判定矩阵

来源：`COMABLE2.ERB`

### 5.1 共同输入

全局判定会反复读取这些变量：

- `ARG`：当前指令编号
- `TARGET`
- `TFLAG:102`
- `TFLAG:100`
- `TFLAG:105`
- `TFLAG:抱住模式`
- `FLAG:潜伏`
- `FLAG:70`
- `CFLAG:TARGET:344`
- `CFLAG:PLAYER:现在忙得抽不出手`
- `TEQUIP:*`
- `ASSIPLAY`

### 5.2 各区间规则

#### `ARG < 270`

解释：

- 属于 `うふふ` / 调教主区间
- 必须在亲密/调教态里执行

核心条件：

- `TFLAG:102 == 2`
- 必须有 `TARGET`
- 套套使用状态会继续限制部分指令

可迁义：

- `erAL` 后续可把这类归成“亲密指令组”

#### `367` 与 `337`

特例：

- `依頼実行`
- `和孩子互动`

特点：

- 允许无 `TARGET`
- 但不能在 `TFLAG:102 == 2/3` 时执行

可迁义：

- 少数指令需要从“必须可见角色”规则里剥离出去

#### `300-303, 305, 331-336, 351, 352, 357, 358, 363-380, 382-399`

解释：

- 日常互动与部分推进指令

核心条件：

- `TFLAG:102 == 1`
- 必须有 `TARGET`
- 潜伏状态下大部分禁用

可迁义：

- 这组非常接近 `erAL` 当前的日常互动、告白、同行、谈心、照料等

#### `306-309`

解释：

- 抱住模式派生指令

核心条件：

- 需要 `TFLAG:抱住模式`
- 或其他特定亲密态

可迁义：

- `erAL` 后续若加“拥抱/贴近”子状态，这组规则很值得直接参考

#### `310-311`

解释：

- 需要 `TARGET`
- 潜伏状态时限制

#### `312-330, 350, 360, 361`

解释：

- 软亲密与部分推进/抵抗指令

核心条件：

- 必须有 `TARGET`
- 不能处在某些 `TFLAG:102` 状态
- 潜伏状态下禁用
- 抱住模式会限制可用子集

可迁义：

- `erAL` 当前的 `tease`、未来的 `kiss`、`lap_pillow`、`confess`，都可以归到这个层级

#### `355`

解释：

- 与时间停止相关

#### `400-499`

解释：

- 单体实行可能指令
- 包括移动、休息、学习、训练、做饭、购物、读书、等待等

核心条件：

- 某些状态下只允许少数指令，例如 `410`、`490`
- 潜伏状态会屏蔽一大批

可迁义：

- `erAL` 的生活指令组应独立于亲密组判定

#### `501-559`

解释：

- `うふふ` 派生
- 需要 `TFLAG:102 == 2`

#### `560-579`

解释：

- 日常派生
- 不能在 `TFLAG:102 == 2`

#### `580-699`

解释：

- 约会指令区

核心条件：

- `TFLAG:100` 必须开启
- 不能处于 `TFLAG:102 == 2/3`
- 潜伏状态禁用

这和 `erAL` 目前的 `requires_date = true` 很像，但 `eraTW` 还多了一层“约会模式总开关”。

#### `700-799`

解释：

- 自慰专用指令

核心条件：

- `CFLAG:PLAYER:现在忙得抽不出手` 等自体状态控制

### 5.3 额外全局过滤器

除了区间外，`GLOBAL_COMABLE` 还会处理：

- 指令过滤器
- 摄影模式
- 游泳状态
- 潜伏模式
- 时间停止
- 套套已使用

这说明 `eraTW` 的可执行性并不是只靠“关系值 + 地点 + 时间”，而是还叠了大量“模式状态”。

## 6. 专用判定样例

### 6.1 `@COM_ABLE86`

文件：`COMABLE_80.ERB`

指令属性：

- 奉仕系

额外条件：

- `!TFLAG:100` 不可
- 过全局判定
- 口球中不可
- 时间停止中不可

结论：

- 即使已经过全局层，单个指令仍可再叠装备/拘束类限制

### 6.2 `@COM_ABLE318`

文件：`COMABLE_300.ERB`

指令属性：

- 日常/软亲密中的更深入动作

额外条件：

- `!TFLAG:100` 不可
- 过全局判定
- 时间停止中不可
- `TCVAR:MASTER:亲热 < 2` 不可
- 着衣状态限制
- 睡眠中限制
- 工作中限制
- 约会中但不在房间时限制
- 陪睡中限制
- 必须特定 `TCVAR` 目标关系
- 气力不能为 0
- 需要 `CFLAG:诶嘿嘿 == 2`

结论：

- 这类指令本质上需要“更细的场景状态”，不只是普通关系阈值

### 6.3 `@COM_ABLE362`

文件：`COMABLE_300.ERB`

指令属性：

- `逃跑`

额外条件：

- 时间停止中不可
- `!TFLAG:100` 不可
- 过全局判定
- 气力不能为 0
- `TFLAG:102 == 4`

结论：

- 抵抗/逃脱类指令依赖专门的流程状态机

### 6.4 `@COM_ABLE611`

文件：`COMABLE_600.ERB`

指令属性：

- 约会中的“买东西”

额外条件：

- 过全局判定
- 约会道中不可
- 睡眠中不可
- 时间停止中不可

结论：

- 约会中还有“进行中/途中/到达”这种细分阶段

### 6.5 `@COM_ABLE612`

文件：`COMABLE_600.ERB`

指令属性：

- 约会中的“送礼”

额外条件：

- 过全局判定
- 必须有 `TARGET`
- 必须处于约会中
- 不能在约会道中
- 睡眠中不可
- 时间停止中不可

结论：

- 约会内指令仍然要叠 `TARGET`、阶段和模式门槛

## 7. 对 erAL 的直接迁移建议

### 7.1 当前欠缺

`erAL` 现在只有一层：

- `available_commands_for_actor()`

而 `eraTW` 实际上至少有三层：

1. 命令组大类判定
2. 指令专用判定
3. 模式过滤器判定

### 7.2 建议补法

先拆成三层，不必一次做完：

1. `CommandCategoryGate`
   - 按日常/同行/约会/轻亲密/房间/自体动作分组
2. `CommandSpecificGate`
   - 每条命令自己的地点、状态、装备、体力、模式判定
3. `GlobalModeGate`
   - 潜伏、途中、存档锁、特殊模式等全局禁用

### 7.3 最值得先迁的语义

- `约会途中不可执行某些指令`
- `亲密深度不足时不能执行更深一级指令`
- `工作中 / 睡眠中 / 潜伏中 / 忙碌中` 这类模式过滤
- 把 `日常 / 软亲密 / 约会 / 自体动作` 做成命令大类
