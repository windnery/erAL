# eraTW Talent 全量编号表

## 1. 目的

这份文档直接保留本地 `eraTW` 的 `CSV/Talent.csv` 全表内容，作为 `erAL` 迁移时的素质词典。

原因：

- `Talent.csv` 不只是简单“编号 -> 名称”。
- 它有大量紧跟条目的注释，直接写明了会怎样影响指令执行、PALAM 上升、ABL 提升、事件结果和口上。
- 如果把它硬拆成摘要表，很容易在整理时丢掉关键语义。

因此这一版采用“原始全表附录”方式保留。

## 4. 使用建议

- 对 `erAL` 来说，优先迁移会直接影响指令前提、数值倍率、关系推进和事件分支的 Talent。
- 不必一口气把全部 Talent 结构化成显式字段，可以先保留“兼容层 + 高优先级显式字段”的做法。
- 后续如果要做更细的迁移，应再拆一份“Talent 直迁优先级排序”文档。
