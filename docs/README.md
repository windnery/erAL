# docs 文档索引

更新时间：2026-04-24

目标：文档只服务当前 `erAL` 重构，不再保留已经失效的 beta 阶段规划口径。

## 1. 当前有效文档

### A 层：执行权威

1. `docs/TODO.md`
   当前重构任务板。先看这个，再动代码。
2. `docs/todo/MVP.md`
   当前 MVP 开发表。用于把已完成底座收口成可玩的最小完整闭环。
3. `docs/系统清单.md`
   当前仓库真实能力、真实规模、真实风险。
4. `docs/PRD.md`
   产品边界和本阶段非目标。

### B 层：辅助理解

1. `docs/玩家体验映射.md`
   系统如何被玩家感知。
2. `docs/TW公式迁移策略.md`
   对 TW 公式模式的分析笔记。仅作迁移参考，不是当前实现真相。

### C 层：子系统规格

1. `docs/specs/README.md`
   规格入口。
2. `docs/specs/*.md`
   子系统说明。当前部分文档仍保留旧字段名和旧命令来源，只能作为参考，不能直接驱动实现。

## 2. 已删除文档

以下文档已删除，因为它们继续保留只会制造噪音：

1. `docs/BETA_REQUIREMENTS_ANALYSIS.md`
2. `docs/BETA_REQUIREMENTS_INTERVIEW.md`
3. `docs/BETA_ROADMAP.md`

原因：

1. 它们面向的是已经失效的 beta 计划阶段。
2. 其中大量数字、入口、命令来源和当前仓库不一致。
3. 这些信息已被当前 `TODO / 系统清单 / PRD` 吸收或替代。

## 3. 当前阅读顺序

1. 先看 `docs/TODO.md`
2. 再看 `docs/系统清单.md`
3. 需要确认边界时看 `docs/PRD.md`
4. 需要理解玩法感知时看 `docs/玩家体验映射.md`
5. 需要参考旧设计或 TW 迁移思路时，再看 `docs/specs/*` 和 `docs/TW公式迁移策略.md`

## 4. 使用规则

1. 如果 `TODO` 和其他文档冲突，以 `TODO` 为准。
2. 如果 `系统清单` 和代码冲突，以代码为准，然后回写文档。
3. `specs` 中凡是提到 `commands.toml` 的地方，默认视为旧口径；当前命令唯一来源是 `data/base/commands/train.toml`。
4. `TW公式迁移策略.md` 中的示例和字段提议，不是当前 schema；只有落进代码和 `TODO` 后才算生效。

## 5. 维护要求

1. 每完成一个阶段性重构任务，必须同步更新 `TODO.md`。
2. 每当运行时契约变化，必须同步更新 `系统清单.md`。
3. 只有当某份 `spec` 与当前代码重新对齐后，才把它从“参考”提升回“受控规格”。
