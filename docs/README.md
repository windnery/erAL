# docs 文档系统总索引

更新时间：2026-04-22

目标：让文档直接服务当前可运行游戏，而不是服务历史阶段计划。

## 1. 快速阅读路径

1. `docs/README.md`：文档边界、权威层级、维护规则。
2. `docs/PRD.md`：当前产品目标、体验边界、验收口径。
3. `docs/玩家体验映射.md`：已实现系统如何被玩家真实感知。
4. `docs/系统清单.md`：已落地系统、数据规模、风险点、下一步补强。
5. `docs/TODO.md`：正在做什么、下个版本做什么。
6. `docs/LONG_TERM_DEV.md`：成品化阶段路线图。
7. `docs/specs/README.md`：按子系统查看详细设计。

## 2. 文档分层

### A 层：执行权威（直接驱动开发）

| 文档 | 用途 | 更新触发 |
|------|------|----------|
| `docs/PRD.md` | 成品目标、验收边界、非目标 | 范围变更、版本切换 |
| `docs/系统清单.md` | 当前真实能力盘点 | 系统新增/下线、数据规模变化 |
| `docs/TODO.md` | 迭代看板、任务状态 | 每次迭代 |
| `docs/LONG_TERM_DEV.md` | 中长期里程碑与阶段策略 | 里程碑切换 |
| `docs/玩家体验映射.md` | 系统到玩家感知的对应关系 | 新系统接入或反馈链变更 |

### B 层：系统规格（受控实现依据）

| 文档 | 用途 |
|------|------|
| `docs/specs/*.md` | 子系统数据结构、流程、配置、扩展约束 |
| `docs/specs/系统总览与缺口.md` | 规格层面的缺口归纳与补强顺序 |

### C 层：参考资料（不可直接作为任务指令）

| 路径 | 用途 |
|------|------|
| `docs/reference/` | eraTW 对照、索引表、迁移模板 |
| `docs/archive/` | 历史方案与已过期规划留档 |

## 4. 冲突处理规则

1. `PRD` 与 `TODO` 冲突时，以 `PRD` 为准。
2. `系统清单` 与具体 spec 冲突时，以代码行为为准，随后同步修文档。
3. 参考层（`reference/`）与实现冲突时，禁止直接按 reference 改代码，必须先更新 A/B 层口径。

## 5. 维护规则

1. 每周至少一次核对 A 层文档口径是否一致。
2. 每次合并跨系统改动后，必须同步更新 `系统清单` 与相关 spec。
3. 新增文档必须归类到 A/B/C 其中一层；未归类文档视为无效文档。

## 6. 删除与归档策略

1. 仍有参考价值但不再执行的文档移入 `docs/archive/`。
2. 内容与当前目标冲突且无历史价值的文档直接删除。
3. 删除文档前，需保证其关键信息已并入 A 或 B 层文档。
