1. 仅为炫技新增系统且无玩家可感知收益的需求。
2. 不带测试与文档的跨系统大改。
3. 与当前产品口径无关的旧阶段实验文档回灌。

## 6. 迭代执行规则

1. 每次迭代至少交付一项玩家可感知改进。
2. 每项任务必须写清 DoD，禁止“做到一半先合并”。
3. 任务关闭前必须更新系统清单和相关 specs。
4. 新增数据字段没有消费者时，不允许进入主分支。
