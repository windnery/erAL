## 🧱 开发规则

1. 跨系统功能落地 `gate + service + data` 三层，不硬写进 UI
2. 前提条件统一放命令门禁，事件层只做触发和表现
3. 道具系统复用 `inventory + item catalog`，不新增平行背包
4. 新系统顺序：`最小数据结构 → 可执行门禁 → 最小闭环 → 内容扩展`
5. 每次会话 `1 主 + 2 次`，任务 25–90 分钟可完成，必须写清 DoD
